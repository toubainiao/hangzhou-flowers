# 🚀 GitHub Pages 部署指南

## 📋 前置要求

- [ ] GitHub账号（没有请去 https://github.com 注册）
- [ ] Git已安装（命令行输入 `git --version` 检查）

## 📝 部署步骤

### 步骤1：创建GitHub仓库

1. 登录 https://github.com
2. 点击右上角 **+** → **New repository**
3. 填写信息：
   - **Repository name**: `hangzhou-flowers`（或你喜欢的名字）
   - **Description**: 杭州赏花日历 - 2025花季指南
   - **Public**（选中）
   - **Add a README file**（不要勾选，我们已有README）
4. 点击 **Create repository**

### 步骤2：上传代码

打开命令行（Windows按 `Win+R` 输入 `cmd`）：

```bash
# 进入项目目录
cd E:\my-pj\hangzhou-flowers

# 初始化Git仓库
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit: 杭州赏花日历网站"

# 添加远程仓库（替换 你的用户名）
git remote add origin https://github.com/你的用户名/hangzhou-flowers.git

# 推送代码
git branch -M main
git push -u origin main
```

### 步骤3：启用GitHub Pages

1. 进入你的GitHub仓库页面
2. 点击 **Settings**（设置）
3. 左侧菜单点击 **Pages**
4. **Source** 部分选择：**Deploy from a branch**
5. **Branch** 选择：**main** / **root**
6. 点击 **Save**

等待2-5分钟，页面会显示：
```
Your site is live at https://你的用户名.github.io/hangzhou-flowers
```

### 步骤4：验证部署

- 点击上面的链接访问
- 确认地图、日历、搜索功能正常

## 🔧 常见问题

### Q1: git push 提示需要登录

**解决**：使用个人访问令牌（PAT）

1. GitHub → Settings → Developer settings → Personal access tokens
2. 生成Token（勾选 repo 权限）
3. 推送时使用Token代替密码

### Q2: 高德地图显示"KEY错误"

**解决**：
- 确认Key已正确填入
- 去高德控制台检查域名白名单
- 添加 `*.github.io` 到白名单

### Q3: 网站样式错乱

**解决**：
- 检查浏览器控制台是否有CORS错误
- 确认所有资源使用HTTPS

### Q4: 如何更新网站内容？

```bash
cd E:\my-pj\hangzhou-flowers
# 修改文件后
git add .
git commit -m "更新内容"
git push
# 等待2分钟后刷新网页
```
